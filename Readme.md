# ToDo App

این یک پروژه وب مبتنی بر جنگو برای مدیریت وظایف (Task Management) است. با استفاده از این اپلیکیشن، کاربران می‌توانند وظایف خود را ایجاد، ویرایش، و مدیریت کنند. این پروژه از قابلیت‌هایی مانند دسته‌بندی وظایف، تعیین اولویت، وضعیت‌های مختلف (برای انجام، در حال انجام، انجام شده)، و تاریخ‌های سررسید و تخمینی بهره می‌برد. تاریخ‌ها با استفاده از `django-jalali` به صورت شمسی نمایش داده می‌شوند.

## ویژگی‌ها

*   **مدیریت کاربران**: هر کاربر می‌تواند وظایف و دسته‌بندی‌های خود را به صورت مجزا مدیریت کند.
*   **دسته‌بندی وظایف**: امکان ایجاد و اختصاص وظایف به دسته‌بندی‌های مختلف برای سازماندهی بهتر.
*   **جزئیات وظیفه**: عنوان، توضیحات، اولویت، وضعیت، مهلت انجام، و زمان تخمینی برای هر وظیفه.
*   **وضعیت‌های مختلف**: وظایف می‌توانند در وضعیت‌های "برای انجام"، "در حال انجام" یا "انجام شده" باشند.
*   **تاریخ‌گذاری شمسی**: استفاده از `django-jalali` برای نمایش تاریخ‌های ایجاد، به‌روزرسانی و تکمیل به صورت شمسی.
*   **بروزرسانی خودکار تاریخ تکمیل**: با تغییر وضعیت وظیفه به "انجام شده"، تاریخ تکمیل به صورت خودکار ثبت می‌شود.

## ساختار پروژه

پروژه از دو بخش اصلی تشکیل شده است:

*   **`core`**: تنظیمات اصلی پروژه جنگو، URLها و سایر فایل‌های پیکربندی.
*   **`task`**: اپلیکیشن اصلی ToDo شامل مدل‌ها (Category و Task)، ویوها، URLها، و فایل‌های ادمین.

## راه‌اندازی و استفاده

برای راه‌اندازی و استفاده از این پروژه در سیستم عامل خود (لینوکس یا ویندوز)، مراحل زیر را دنبال کنید:

### پیش‌نیازها

*   **پایتون 3.x**: اطمینان حاصل کنید که پایتون 3 روی سیستم شما نصب شده است.
    *   برای بررسی: `python --version` یا `python3 --version`
*   **pip**: ابزار مدیریت پکیج پایتون.
    *   برای بررسی: `pip --version` یا `pip3 --version`

### مراحل راه‌اندازی

1.  **کلون کردن مخزن (یا دانلود پروژه)**:
    اگر پروژه در یک مخزن Git قرار دارد، آن را کلون کنید:
    ```bash
    git clone https://github.com/Nikbonyad/ToDoApp.git
    cd ToDoApp
    ```
    در غیر این صورت، فایل‌های پروژه را دانلود کرده و در یک پوشه قرار دهید و سپس وارد آن پوشه شوید.

2.  **ایجاد و فعال‌سازی محیط مجازی (توصیه می‌شود)**:
    ایجاد محیط مجازی به شما کمک می‌کند تا وابستگی‌های پروژه را از سایر پروژه‌های پایتون جدا نگه دارید.

    **در لینوکس/macOS:**
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    ```

    **در ویندوز:**
    ```bash
    py -m venv venv
    .\venv\Scripts\activate
    ```

3.  **نصب وابستگی‌ها**:
    پس از فعال‌سازی محیط مجازی، تمام پکیج‌های مورد نیاز پروژه را از فایل `requirements.txt` نصب کنید:
    ```bash
    pip install -r requirements.txt
    ```

4.  **اعمال مهاجرت‌های پایگاه داده**:
    این مرحله مدل‌های جنگو شما را به ساختار پایگاه داده تبدیل می‌کند.
    ```bash
    python manage.py makemigrations
    python manage.py migrate
    ```

5.  **ساخت سوپریوزر (اختیاری اما توصیه می‌شود)**:
    برای دسترسی به پنل ادمین جنگو، نیاز به یک کاربر با دسترسی سوپریوزر دارید:
    ```bash
    python manage.py createsuperuser
    ```
    اطلاعات کاربری (نام کاربری، ایمیل، رمز عبور) را وارد کنید.

6.  **اجرای سرور توسعه**:
    اکنون می‌توانید سرور توسعه جنگو را راه‌اندازی کنید:
    ```bash
    python manage.py runserver
    ```
    پس از اجرای موفقیت‌آمیز، پیامی مشابه زیر در کنسول مشاهده خواهید کرد:
    ```
    Watching for file changes with StatReloader
    Performing system checks...

    System check identified no issues (0 silenced).

    January 26, 2026 - 14:00:00
    Django version 5.x, using settings 'core.settings'
    Starting development server at http://127.0.0.1:8000/
    Quit the server with CONTROL-C.
    ```

7.  **دسترسی به اپلیکیشن**:
    مرورگر وب خود را باز کنید و به آدرس `http://127.0.0.1:8000/` بروید.
    برای دسترسی به پنل ادمین (در صورتی که سوپریوزر ساخته‌اید) می‌توانید به `http://127.0.0.1:8000/admin/` مراجعه کنید.

### توقف سرور

برای توقف سرور توسعه، کافیست در پنجره ترمینال یا Command Prompt که سرور را اجرا کرده‌اید، کلیدهای `Ctrl + C` را فشار دهید.
